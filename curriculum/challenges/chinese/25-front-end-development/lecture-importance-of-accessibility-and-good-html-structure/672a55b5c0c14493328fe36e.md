何时需要 `alt` 属性，以及好的 `alt` 文本有哪些示例？

替代文本，通常缩写为 `alt` 文本，是对图片的简要文字描述。它为无法看到图片的用户（如使用屏幕阅读器和其他辅助技术的人）提供了图片的关键信息。

这对于让网站对视力障碍者更友好至关重要。

替代文本还被搜索引擎用于理解图片内容。有些浏览器在图片无法正常加载时也会显示它，比如图片文件丢失或用户网络不佳时。因此，alt 文本有多种用途。

视力障碍者如果没有 alt 文本的描述，将无法知道图片内容。例如，一张小狗的图片，如果 alt 文本写成“可爱的小狗”，这就是一个不好的例子，因为它没有具体描述图片的关键信息，比如小狗长什么样？小狗在哪里？周围是否有重要的物体？

让我们改进一下。一个好的替代文本示例可以是：“一只戴着橙色项圈的黑白小狗趴在沙滩上，侧头望向一旁。它前爪旁边有一个亮橙色的球。”

你可以在 HTML 中这样使用 image 元素、alt 属性和更详细的描述：

```html
<img src="puppy.png" alt="一只戴着橙色项圈的黑白小狗趴在沙滩上，侧头望向一旁。它前爪旁边有一个亮橙色的球。" />
```

需要注意的是，alt 文本的写法没有唯一标准。你在描述中包含哪些内容取决于图片的使用场景。例如，如果这张小狗的图片用于介绍犬种的网站，你可能需要更详细地描述小狗的外貌，而可以忽略小狗在沙滩上玩橙色球的细节。最终，图片的 alt 文本应反映图片在页面上的主要用途，并为无法看到图片的人提供理解该用途所需的信息。

再举一个热带度假村的例子。我们来描述一下。

这个图片如果 alt 文本写成“度假村”，就是一个不好的例子。

这个描述太短，无法提供足够的信息。可以这样改进：

“热带度假村，设有被棕榈树和小屋环绕的游泳池。”

你可以这样在 HTML 中使用 alt 属性：

```html
<img src="resort.png" alt="热带度假村，设有被棕榈树和小屋环绕的游泳池。" />
```

现在你已经了解了好的和不好的 alt 文本，下面是一些最佳实践：

- alt 文本应尽量简短，既要详细到能理解图片内容，又不要太长导致混淆。
- 不需要描述每一个细节，重点突出图片最重要的方面。
- 通常不需要以“图片”或“照片”开头，可以直接描述内容。
- 如果图片周围有类似的文字，就不需要在 alt 文本中重复。
- 建议 alt 文本以句号结尾，保持一致性。
- 如果图片是链接，alt 文本应描述点击后会发生什么，而不是图片本身。

例如，网站上有一个右箭头图标，点击后跳转到下一页。如果 alt 文本只写“右箭头”，如下：

```html
<a href="about.html">
  <img src="arrow-right.png" alt="右箭头。" />
</a>
```

更好的做法是让 alt 文本描述点击后的结果：

```html
<a href="about.html">
  <img src="arrow-right.png" alt="进入下一页。" />
</a>
```

只有传递重要信息的图片才需要 alt 文本。如果图片仅用于装饰，应将 alt 属性设为空（即空字符串），这样屏幕阅读器和辅助技术会忽略它。

例如：

```html
<img src="decorative_image.jpg" alt="" />
```

你网站上的每张图片都应有 alt 属性，即使是空的。如果完全省略 alt 属性，有些屏幕阅读器会读出文件名，这会干扰使用辅助技术的用户，因此不推荐这样做。

最后，在网站上线前，应仔细测试屏幕阅读器能否正确读取 alt 文本。

编写有效的 alt 文本对于创建无障碍网页内容至关重要。作为开发者，通过为图片提供清晰的描述，可以确保每个人都能平等地访问和使用你的网站。
